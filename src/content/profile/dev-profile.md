## Agent 设计理解

> source id: `profile.agent`

我对 Agent 的核心判断是：Agent 不是更会聊天的机器人，而是围绕目标、上下文、工具调用、执行、验证和权限边界构建的工作系统。

一个可用 Agent 至少要能理解任务目标、读取上下文、选择工具、执行动作、检查结果，并在权限边界内和人协作。判断产品是不是 Agent，重点看它能不能交付可验证、可追踪的结果，而不是只输出建议。

## Skills 与 MCP 判断

> source id: `profile.skills_mcp`

我把 Skills 和 MCP 当成工程 Agent 的能力边界管理方式，而不是越多越好的插件列表。

Skills 更像“按场景加载的专业工作方式”，MCP 更像“让 Agent 接触外部系统的权限通道”。是否启用，取决于角色、触发场景、权限范围、风险和维护成本。
