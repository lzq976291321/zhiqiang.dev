# SJ 阶段项目集

> source id: `profile.sj`

这份资料用于回答我在 SJ 阶段做过哪些项目、这些项目应该如何公开展示、以及它们体现了哪些开发能力。

对外展示时，应把这些经历抽象成产品域和能力域，不主动暴露内部仓库路径、内部 Git 地址、配置文件、私有 npm 文档、接口细节或未公开业务数据。

## 总览

> source id: `profile.sj_overview`

SJ 阶段覆盖 20+ 个前端相关项目，核心围绕视频创作、直播、数字人、模板市场、链接成片、业务后台、官网落地页、渠道平台、桌面端、移动端小程序，以及内部前端基建。

这段经历最适合对外呈现为：复杂视频创作产品线的前端工程化实践，而不是一组零散仓库清单。

## 视频创作工作台与微前端应用群

> source id: `profile.sj_workbench`

主要项目包括工作台基座、业务后台、图文/口播视频、链接成片、模板市场、定制数字人、商城、官网和相关公共导航/登录模块。

这组项目体现的能力包括：

- React + TypeScript + MobX + Vite 的大型业务前端开发。
- 微前端基座与多个业务子应用协作。
- 素材上传、视频生成、任务状态、模板选择、付费/订阅、导航登录等横向能力复用。
- 面向创作者的内容生产链路：从素材输入、模板选择、生成任务、状态反馈到发布管理。

## 视频编辑器与内容生产核心

> source id: `profile.sj_editor`

视频编辑器和内容生产是 SJ 阶段最核心的项目经验之一。公开简历中可表述为：主导过视频编辑器主体架构设计和几何算法核心模块，支撑裁剪、拼接、特效叠加、字幕轨道、实时预览、Undo/Redo、多轨道协同等能力。

相关工程能力包括：

- 发布订阅解耦编辑器模块通信。
- 策略模式处理不同媒体渲染。
- 工厂模式统一素材创建。
- 装饰器模式做无侵入扩展。
- WebSocket 替代轮询，提升任务状态反馈实时性。
- 分片上传、视频处理状态同步和批量任务体验优化。

## 数字人、直播与桌面端

> source id: `profile.sj_live_digital`

相关项目覆盖定制数字人、AI 直播/智播系统、BocaLive 官网和 Electron 桌面端。

这组经历可以说明我不只做网页页面，还处理过更重的音视频和桌面端场景，包括 Electron、React、MobX、Vite、音频播放器、快捷键、Socket、状态管理、国际化、桌面端打包和错误监控等工程问题。

## 前端基建与内部工具链

> source id: `profile.sj_infra`

SJ 阶段沉淀过多类内部前端基建：

- Lithe UI：轻量 UI 组件库。
- Lithe Animation：基于 GSAP 的动画库。
- utils：内部通用工具函数库。
- app-types：公共数据模型类型包。
- mtv-navbar / mtv-login：导航、登录、支付和账号相关公共模块。

这部分适合展示“从业务开发抽象出公共能力”的经验：组件库、工具库、动画库、公共类型、登录导航、构建发布、私有 npm 包维护和团队规范。

## 多端与增长产品

> source id: `profile.sj_multi_end`

除 React/Vite 主线项目外，SJ 阶段还覆盖过 Next.js 官网/落地页、Vue 管理端、Taro 小程序、Arco/Ant Design 后台、渠道平台和商城等项目。

这组经历适合回答“我是否能适应不同技术栈和业务形态”：能在 React、Next.js、Vue、Taro、Electron 等不同前端栈之间切换，同时保持 TypeScript、工程化、状态管理、构建部署和业务交付的一致思路。
