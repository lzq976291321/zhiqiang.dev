Linear MCP 的价值在于让 Agent 从真实 issue 开始工作，而不是靠你复制粘贴需求。

适合的流程：

- 从 issue 读取背景和验收标准
- 拆开发任务
- 更新实现进度
- 关联 PR 和修复说明

写权限要谨慎开放，避免 Agent 自动污染任务状态。
