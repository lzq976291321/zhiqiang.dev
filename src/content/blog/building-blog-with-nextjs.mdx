## 为什么要自己搭

用过 Notion、语雀、掘金，最终还是决定自己搭。原因很简单：

1. **完全控制设计** — 想要什么风格就做什么风格
2. **MDX 很强大** — Markdown 里嵌入 React 组件
3. **SEO 友好** — Next.js SSG 天然支持
4. **学习驱动** — 做个人项目是最好的学习方式

## 技术选型

| 层 | 选择 | 理由 |
|---|---|---|
| 框架 | Next.js 14 App Router | SSG + ISR |
| 样式 | Tailwind CSS v4 | 原子化，暗色主题友好 |
| 动画 | Framer Motion | React 生态最佳 |
| 内容 | MDX + gray-matter | 本地文件，零依赖 |
| 代码高亮 | Shiki | Tokyo Night 主题 |
| 部署 | Vercel | 零配置 |

## 核心实现

### MDX 解析

用 `gray-matter` 解析 frontmatter，`next-mdx-remote` 做服务端渲染：

```typescript
import matter from "gray-matter"

export function getMdxFiles(dir: string) {
  return fs.readdirSync(fullPath)
    .filter(f => f.endsWith(".mdx"))
    .map(filename => {
      const raw = fs.readFileSync(filePath, "utf-8")
      const { data, content } = matter(raw)
      return { slug, frontmatter: data, content }
    })
}
```

### 动画规范

所有动画统一用 Framer Motion，定义基准参数避免每次重写：

```typescript
const fadeInUp = {
  initial: { opacity: 0, y: 30 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.6, ease: [0.16, 1, 0.3, 1] }
}
```
