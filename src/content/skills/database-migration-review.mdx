数据库迁移不是把 schema 改到能跑就结束。

审查重点：

- 是否有可执行回滚方案
- 是否会长时间锁表
- 是否需要分阶段迁移
- 索引是否匹配查询路径
- RLS / 权限是否随 schema 同步更新
- 数据修复脚本是否可重复执行

对独立开发者来说，这个 Skill 能避免很多“上线后才发现数据坏了”的问题。
