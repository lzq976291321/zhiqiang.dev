API 设计最容易被低估的是长期维护成本。

这个 Skill 重点检查：

- 请求和响应是否稳定
- 错误码是否能指导调用方恢复
- 是否需要分页、排序、过滤
- 写操作是否幂等
- 权限和审计是否明确
- 是否暴露了内部实现细节

Agent 调用 API 时尤其需要结构化错误，否则模型只能猜下一步。
